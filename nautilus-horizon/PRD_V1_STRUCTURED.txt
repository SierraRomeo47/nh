4. FUNCTIONAL REQUIREMENTS
================================================================================

4.1 AUTHENTICATION & AUTHORIZATION
================================================================================

AUTHENTICATION REQUIREMENTS:
├── Multi-Factor Authentication (MFA)
│   ├── Required for: ADMIN, MANAGER, COMPLIANCE_OFFICER
│   ├── Optional for: All other roles
│   └── Implementation: TOTP, SMS, Email verification
├── Session Management
│   ├── Session timeout: 8 hours (admin), 12 hours (operational)
│   ├── Concurrent session limit: 2 per user
│   └── Session invalidation on role change
├── Password Policy
│   ├── Minimum 12 characters
│   ├── Complex character requirements
│   ├── Password history: 12 previous passwords
│   └── Password expiration: 90 days
└── Account Lockout
    ├── Failed attempts: 5 before lockout
    ├── Lockout duration: 30 minutes
    └── Admin unlock required after 3 lockouts

AUTHORIZATION FRAMEWORK:
├── Role-Based Access Control (RBAC)
│   ├── Hierarchical permission inheritance
│   ├── Dynamic role assignment
│   └── Permission delegation capabilities
├── Attribute-Based Access Control (ABAC)
│   ├── Context-aware permissions
│   ├── Time-based access controls
│   └── Location-based restrictions
├── Data Access Controls
│   ├── Row-level security by vessel/ship
│   ├── Column-level security for sensitive data
│   └── Field-level encryption for PII
└── API Access Controls
    ├── OAuth 2.0 with JWT tokens
    ├── Rate limiting per role
    └── API key management for external systems
